# 할 일 페이지 인터랙션 명세

> Claude in Chrome MCP로 실제 NRS 사이트(`/nrs/working-station/tasks`)에서
> 직접 클릭/드래그/키 입력을 시도해 관찰한 결과.
> 이 문서는 React 프로토타입(`src/pages/Tasks.tsx`, `TaskItem.tsx`, `WeeklyCalendar.tsx`)에
> 1:1 재현 또는 V1/V2 변형 시 참조하는 단일 출처.

---

## 요약: 가장 빈번할 인터랙션 TOP 5

| 순위 | 인터랙션 | 빈도 가설 | 프로토타입 우선순위 |
|---|---|---|---|
| 1 | 업무 제목 클릭 → 우측 사이드 패널 열림 | 매번 | **P0** |
| 2 | "+ 업무 추가" → 인라인 인풋 → Enter 연속 입력 | 일상 | **P0** |
| 3 | 행 인라인 컨트롤(상태/우선순위/마감일) 클릭 → 드롭다운/팝오버 | 자주 | **P0** |
| 4 | 캘린더 빈 슬롯 클릭 → 해당 시간대 업무 빠른 추가 | 매일 | **P1** |
| 5 | "하루 정리하기" 모달 (성과 입력 → 그라운드룰 → 정리) | 매일 1회 | **P1** |

**프로토타입 V0(원본 재현)에 P0 3개를 먼저 넣으면 데모로 충분.**
V2/V3에서 캘린더-업무 연결을 새로 설계할 때 위 4번 인터랙션이 핵심 진입점.

## 영역별 상세

### 1. 상단 헤더

| 트리거 | 동작 | 네트워크 | 비고 |
|---|---|---|---|
| "완료 목표, 업무" 버튼 클릭 | 팝오버 오픈, 토글 3개(완료 목표 포함, 완료 업무 포함, 그 외 1개) | (확인 못함) | 토글만 클릭 시 자동 닫힘이 아니라 **반드시 [적용] 버튼 필요** |
| [적용] 버튼 클릭 | 팝오버 닫힘 + 리스트 갱신 | API 호출 추정 | 배경 클릭으로는 닫히지 않음 |
| "업무 집중 모드" 버튼 | **별도 페이지 `/tasks-space`로 이동** | navigation | 팝오버/모달 아님 — 풀스크린 라우트 |

### 2. 오늘의 업무 추천

| 트리거 | 동작 | 비고 |
|---|---|---|
| 섹션 헤더 ▼ 클릭 | 접힘 ▶, 내용 사라짐 / 다시 클릭하면 펼침 | 상태가 로컬 |
| 비어있을 때 | "오늘은 추천할 업무가 없어요." + 🗂️ 아이콘 | **추천 알고리즘은 미공개** → V1에서 우리 가설(영향력)로 대체 |

### 3. 목표별 업무 섹션

| 트리거 | 동작 | 네트워크 |
|---|---|---|
| 섹션 헤더 ▼ 클릭 | 섹션 접힘/펼침 (▼ ↔ ▶) | - |
| "+ 업무 추가" 클릭 | **섹션 최상단**에 인라인 인풋 등장. placeholder: "목표에 기여할 업무를 등록해 주세요.", [취소][추가] 버튼 | - |
| 인풋에서 Enter (빈 값) | 아무 일 없음, 인풋 유지 | - |
| 인풋에서 Esc | 인풋 사라짐 (취소) | - |
| 인풋에서 Enter (값 있음) | 업무 추가 + **카운터 0/1 → 0/2 증가** + 인풋 유지 → **연속 입력 가능** | POST 추정 |

> 카운터 `n/m` = (완료 수) / (전체 수). "0/1" 상태에서 추가하면 "0/2"가 되는 것으로 확인.

### 4. 개별 업무 행 (TaskItem) — 가장 정보 밀도 높음

| 트리거 | 동작 | 비고 |
|---|---|---|
| **제목 클릭** | 우측 슬라이드인 사이드 패널 ("업무 상세") 열림 | **P0** |
| 제목 옆 ✏️ (편집 아이콘) 클릭 | 행이 인라인 편집 모드로 전환 (인풋 + [취소][저장]) | 호버 시 노출 |
| 호버 시 우측 끝 🗑️ (삭제) | **확인 모달**: "이 업무를 삭제하시겠어요?" + [취소][확인] | 호버 의존 |
| 우클릭 | 컨텍스트 메뉴 **없음** (브라우저 기본도 차단됨) | - |
| 체크박스 mark 클릭 | **이 mark는 호버 시에만 시각적으로 보이며 클릭 핸들러가 없음** → 완료 처리는 상태 드롭다운 "완료" 선택으로만 가능 | ⚠️ 발견 |
| 행 내 "대기/진행/완료" 클릭 | 드롭다운 (대기✓ / 진행 / 완료) | 실제로 작동 |
| 행 내 마감일 📅 클릭 | 달력 팝오버 (월/연 이동 화살표, 오늘 하이라이트) | - |
| 행 내 우선순위 (낮음/보통/높음) 클릭 | 드롭다운 (높음 / 보통✓ / 낮음) | - |
| 행 내 "🏆 성과 등록" 클릭 | 모달 오픈 — 성과 내용 textarea (0/1000) + 첨부파일 + [취소][확인] | - |
| 하위 업무가 있는 행의 ▶ 클릭 | 하위 업무 인라인 펼침 (트리 확장) | 발견된 인터랙션 |
| 사이드 패널의 상태 클릭 | 패널 내에서도 동일한 드롭다운 가능 | 행 vs 패널 동작 동일 |
| Esc | 사이드 패널 닫힘 | - |

**사이드 패널 필드 목록** (재현용):
- 연결된 목표 / 상태(대기/진행/완료) / 업무 유형(일반 업무) / 담당자 / 요청자 / 시작일 / 종료일 / 중요도(낮음/보통/높음) / 성과 수치 / 업무 목표 textarea / 업무 메모 및 성과 textarea / 첨부 파일 / 하위 업무 / 체크리스트 / 이슈 / 댓글

### 5. 우측 주간 캘린더

| 트리거 | 동작 | 비고 |
|---|---|---|
| 날짜 셀(12~18) 클릭 | 해당 날짜로 전환, 그날 일정만 표시 (12일 클릭 시 일정 없음 표시) | 단일 날짜 모드인지 주간 유지인지 재확인 필요 |
| "2026.04" 텍스트 클릭 | **반응 없음** — 월 피커 없음 | - |
| 좌/우 화살표(`<` `>`) 클릭 | 이전/다음 주 이동 (현재 주 기준 4/5~4/11 ↔ 4/12~4/18 ↔ 4/19~4/25) | 화살표는 캘린더 우상단 영역. JS 클릭으로만 확인됨 (좌표 클릭 어려움) |
| 시간 슬롯 빈 영역 클릭 | **해당 섹션 상단에 업무 추가 인풋 등장** (캘린더에 직접 일정 생성이 아니라 업무 리스트에 추가됨) | ⚠️ 흥미로운 발견 |
| 시간 슬롯 빈 영역 드래그 | (확인 못함) | - |
| 기존 일정 블록 클릭 | **해당 업무의 상태 드롭다운만** 열림 (사이드 패널 아님) | ⚠️ 행 클릭과 다른 동작 |
| 일정 블록 드래그 / 리사이즈 | (확인 못함) | - |
| "🗒 하루 정리하기" 클릭 | **풀스크린 모달, 3-step 위저드** | **P1** |

**하루 정리 위저드 3 스텝**:
1. **성과 작성하기** — 오늘 잡힌 모든 업무가 리스트로 나오고 각각 상태 드롭다운 + 성과 입력 필드. [다음]
2. **그라운드룰** — 등록된 룰이 있으면 표시, 없으면 "그라운드룰이 없어요." [이전][다음]
3. **하루 정리하기** — (내용 미확인)
- 모달은 **Esc / 배경 클릭으로 닫히지 않음**. X 버튼만 있음 (위치 재확인 필요). ⚠️ 페이지 새로고침으로 강제 종료됨

## 빈 상태 / 로딩 / 에러

- **추천 빈 상태**: "오늘은 추천할 업무가 없어요." + 🗂️
- **그라운드룰 빈 상태**: "그라운드룰이 없어요. 그라운드룰이 등록된 경우 확인할 수 있어요."
- **다음 주 일정 빈 상태**: 시간 슬롯만 있고 블록 없음
- **삭제 확인 모달**: "이 업무를 삭제하시겠어요?" / [취소][확인]
- **목표 미연결 섹션**: 일반 목표 섹션과 동일하게 동작 (별도 인터랙션 차이는 발견 안 됨)

## 프로토타입 재현 시 미해결 질문

1. **"오늘의 업무 추천" 알고리즘 미공개** → V1에서 우리 가설(`진행률 갭 × 마감 임박도`)로 대체 (이미 구현 완료, `/tasks/v1`)
2. **체크박스 mark가 시각만 있고 핸들러 없음** — 완료 처리가 정말 상태 드롭다운 경유만인지 확인 필요. 만약 그렇다면 우리 V0에서는 mark를 클릭 가능하게 만드는 게 오히려 개선이 됨 (V2 후보)
3. **캘린더 빈 슬롯 클릭 = 업무 리스트에 추가** — 시간/날짜 정보가 없는 일반 추가가 됨. 이건 명백한 UX 결함으로 V2 후보 ("슬롯에 드롭하면 그 시간으로 잡힘"이 자연스러움)
4. **캘린더 일정 블록 클릭 = 상태 드롭다운만** — 사이드 패널이 안 열려 어떤 업무인지 맥락 부족
5. **"하루 정리" 모달이 Esc/배경 클릭으로 안 닫힘** — 명백한 모달 트랩 (V2 후보)
6. **드래그 인터랙션은 시도 못함** — 행 ⠿ 핸들 / 일정 블록 / 리사이즈 모두 미확인. Chrome MCP의 드래그 한계
7. **단축키 시스템 없음** — `?`, `/`, `Cmd+K` 모두 미작동. V2에서 추가 가치 큼

## V2/V3 변형 후보 (인터랙션 기반 개선)

발견된 결함을 바탕으로 한 다음 실험 후보:

### V2-A: "캘린더 슬롯 = 시간 지정 추가"
**가설**: 빈 슬롯 클릭이 시간 정보 없이 추가되는 건 직관과 어긋난다.
**개선**: 슬롯 클릭 시 인풋 + 시간/길이 자동 채움 → Enter 시 즉시 캘린더 블록 생성.

### V2-B: "체크박스 1-클릭 완료"
**가설**: 완료 처리에 드롭다운 2-클릭이 필요한 건 마찰이 크다.
**개선**: mark를 실제 클릭 가능한 체크박스로. 클릭 1번에 완료(낙관적 UI) + 5초 Undo 토스트.

### V2-C: "캘린더 블록 클릭 = 사이드 패널"
**가설**: 캘린더에서 일정을 클릭하면 그 업무의 맥락(목표, 마감, 우선순위)을 보고 싶은 게 자연스럽다.
**개선**: 블록 클릭 시 상태 드롭다운 대신 사이드 패널. 상태 변경은 패널에서.

### V2-D: 단축키 시스템
**가설**: 매일 쓰는 도구는 키보드가 답이다.
**개선**: `n` (새 업무), `/` (검색), `j/k` (행 이동), `space` (완료 토글), `Esc` (패널 닫기).

### V3: "캘린더-업무 양방향 자동 정리"
- V1(영향력 추천) + V2-A(슬롯 지정) + 자동 빈 슬롯 채움(`findEarliestSlot`)을 하나로 통합.
- "오늘 잡혀야 할 핵심 업무 N개"를 한 번에 캘린더에 자동 배치하는 [모두 잡기] 버튼.

## 메모: 기술적 관찰

- 사이트는 **CSS 모듈** 기반 (`goalItem-module__btnDeleteTask--qA8lQ` 등). 클래스 이름이 빌드별로 바뀌는 해시를 가짐 → 셀렉터 의존 자동화는 깨지기 쉬움.
- 캘린더 화살표 등 일부 컨트롤은 **z-index/포인터 이벤트 문제**로 좌표 클릭이 어려움. JS evaluate가 더 안정적.
- "업무 집중 모드"는 별도 라우트(`/tasks-space`) — 우리 프로토타입에 같은 라우트로 추가 가능.
- 하위 업무가 있는 행의 ▶ 펼침은 트리 구조 (Goals 페이지 트리와 유사) — 재현 시 컴포넌트 재사용 가능.
